### 目前思路是将音频的梅尔频谱(灰度图)转换成RGB图,与CIFAR-10对比训练

根据实验结果,发现在高epoch的情况下clean_dataset的正确率较低,即不可学习的效果较好.
当然,我也发现生成的噪音中有大量的无效噪音(noise为0的部分),遂修改了Generate Error-Minimizing Noise中的noise参数,noise = torch.zeros([50000, 3, 32, 32])修改为noise = torch.zeros([800, 3, 32, 32]),生成的噪音只有一组0.于是我猜测生成的噪音与第一个参数有关,并且这个参数与训练集数量有关,而我的训练集的理论训练集数量刚好是800,仍有一组噪音的原因可能是训练集有部分损坏.在Preparation部分对数据集进行了验证,发现部分类别的训练数量并没有达到80(共10类,每类理论上80个训练,共800个).

关于在高epoch情况下不可学习效果较好的原因,我猜测是数据集内数据太少,导致需要几轮的训练才能让它真正拟合进去,在epoch<75时,效果较差,epoch在[100,200]范围内效果较好.当然在更大的范围内也许有更适合的epoch值使得实验达到更完美的效果,在此并没有去探寻这个问题.

GTZAN数据集内有png图,可以根据png图进行训练,进而达到目的.因在训练时并未注意到GTZAN/GTZAN/images_original内的png图片而直接对音频进行训练,后续可以在png的基础上训练,或许效果更好.

故当前思路的改进方案为:增加数据集验证在大量数据训练情况下效果是否更好;使用转换好的png进行训练;修改部分参数使之更加适配音频不可学习方案.
